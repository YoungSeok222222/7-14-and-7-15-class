# Vue Router 의 개념

## Router 란 ?

- 네트워크에서의 라우터
  - 컴퓨터 네트워크 간에 데이터 패킷를 전송하는 네트워크 장치
  - 패킷의 위치를 추출하여, 그 위치에 대한 최적의 경로를 지정
    -> 이 경로를 따라 데이터 패킷을 다음 장치로 전달
- 웹에서의 라우터
  - 클라이언트의 요청 경로를 보고 이 요청을 처리할 수 있는 곳으로 기능을 전달해주는 역할

## Vue-Router

- Vue.js 공식 플러그인으로 제공되는 라우팅 라이브러리
- SPA(Single Page Application) 개발에 사용
  - SPA 방식
    - 최초 접속 시 모든 CSS, JS 파일을 가져와서 페이지를 구성
    - 필요 시 서버의 REST API 데이터를 비동기 요청하여 받아와서 렌더링
      - 화면 자체를 서버에서 받아오는 것이 아니라, 클라이언트에서 렌더링(CSR)
    - 화면 구성에 필요한 소스들을 최초 접속 시 받아왔기 때문에 클라이언트가 가지고 있음
      - SPA 방식이 기존 방식보다 반응이 빠르다
      - 최초 접속 시에는 느릴 수 있다.

- Vue 는 SPA 방식으로 개발된다.
  - 한 페이지는 여러 개의 Vue Component 로 구성되어 있다
  - 즉, `다른 화면을 보여준다 == 다른 Vue Component 를 보여준다` 라는 뜻
  - 이 때, 화면 간에 전환을 매끄럽게 도와주는 것이 __Vue Router__
  - 프론트엔드에서 요청 URI 에 따라 전체 DOM 을 변경하는 것이 아니라, 브라우저에서 변화가 있는 부분의 DOM 만 변경하는 방식을 지원해준다.

- 프로젝트에 Vue Router 추가하는 방법
`$ vue add router`

- 사용법
1. URL -> View 라우팅 설정
  - `<project>/router/index.js` 에 존재

2. Component 에서 사용법
  - URL : `<router-link>`
  - View : `<router-view>
