무결성 제약조건(Integrity Constraint)
  - DB의 무결성을 보장하기 위해 저장, 삭제, 수정 등을 제약하기 위한 조건

무결성 제약조건은 크게 3가지가 + a 존재

1. 개체 무결성
  - 모든 테이블이 기본 키(primary key)로 선택된 필드(column)를 가져야 한다.
  - 기본 키로 선택된 필드를 고유한 값을 가져야 하며, 빈 값은 허용하지 않는다.
            UNIQUE            NOT NULL
 
2. 참조 무결성
  - 참조 관계에 있는 두 테이블의 데이터가 항상 일관된 값을 갖도록 유지하는 것
  - 외래키 값은 NULL 이거나 참조 테이블의 기본키 값과 동일해야 한다.
    -> 즉, 테이블은 참조할 수 없는 외래키 값을 가질 수 없다.
  - 외래키에 NULL 값도 가능하다.
    - ex ) 사원은 하나의 부서를 갖는다. 하지만, 신입사원은 부서가 미정일 수 있다.

3. 범위(=도메인 =정의역) 무결성
  - 테이블에 존재하는 필드의 무결성을 보장하는 위한 것
  - 필드의 타입, NULL 값 허용 등에 대한 사항을 정의하고, 올바른 데이터가 입력되었는 지 확인
  - 즉, 걸어 놓은 제약 조건에 위반하지 않는 데이터만 입력받겠다 !

4. 추가(무결성 규칙)
  - 규칙에 따른 무결성을 보장해야 한다.
  - NULL 무결성 : 속성 값은 NULL 로 설정되면, NULL 값은 못 들어온다.
  - 고유 무결성 : UNIQUE 값 설정 시 각 레코드들이 갖는 값이 서로 달라야 한다.
  - 키 무결성 : 하나의 테이블에는 적어도 하나의 키가 존재해야 한다. ( 기본키, 외래키, 후보키 등 )
  - 관계 무결성 : 테이블의 어느 한 레코드의 삽입 가능 여부 또는
    한 테이블과 다른 테이블의 레코드들 사이의 관계에 대한 적절성 여부를 보장해야 한다.
